# Planning Directory

This directory contains planning documents, project management files, and strategic documentation for the Oracle Forge project.

## Purpose

The planning directory serves as the central hub for:
- **Project strategy** and roadmap
- **Task breakdown** and assignment
- **Milestone tracking** and progress reporting
- **Decision documentation** and rationale

## Files Overview

### Strategic Documents
- **Project roadmap** - Long-term vision and milestones
- **Architecture decisions** - Technical decisions and trade-offs
- **Risk assessment** - Potential risks and mitigation strategies

### Task Management
- **Task breakdown** - Detailed work items and dependencies
- **Assignment tracking** - Who is working on what
- **Progress reports** - Status updates and completion tracking

### Meeting Notes
- **Sprint planning** - Iteration planning and goals
- **Retrospectives** - Lessons learned and improvements
- **Stakeholder updates** - Communication with project stakeholders

## How Planning Integrates with Development

### Before Development
1. **Define requirements** in planning documents
2. **Break down tasks** into manageable work items
3. **Assign ownership** and set deadlines
4. **Identify dependencies** and potential blockers

### During Development
1. **Track progress** against planned milestones
2. **Update documentation** as decisions are made
3. **Log issues** and solutions for future reference
4. **Adjust plans** based on new information

### After Development
1. **Document outcomes** and lessons learned
2. **Update processes** based on retrospectives
3. **Archive completed work** for future reference
4. **Plan next iterations** based on results

## File Organization

### Naming Conventions
- **Dates first**: `2025-04-18_sprint_planning.md`
- **Clear purpose**: `architecture_database_decisions.md`
- **Version control**: `v2_requirements.md`

### Document Structure
Each planning document should include:
- **Purpose** - Why this document exists
- **Scope** - What is covered and what is not
- **Decisions** - Key decisions and rationale
- **Actions** - Next steps and owners
- **Status** - Current state (draft, approved, archived)

## Best Practices

### Documentation
- **Keep documents current** - Update as things change
- **Be concise** - Use bullet points and clear language
- **Include context** - Help others understand the background
- **Link related docs** - Cross-reference relevant information

### Process
- **Review regularly** - Ensure plans reflect reality
- **Get feedback** - Involve team members in planning
- **Document changes** - Track why decisions were made
- **Archive old work** - Keep current plans focused

## Integration with Other Directories

- **`kb/`** - Planning decisions inform knowledge base
- **`agent/`** - Technical specs guide implementation
- **`eval/`** - Planning defines success criteria
- **`signal/`** - Planning outcomes communicated externally

## Access and Permissions

Planning documents may contain:
- **Sensitive information** - Access restricted as needed
- **Draft decisions** - Not for external distribution
- **Personal notes** - Individual thoughts and drafts

Be mindful of classification levels and distribution rules.
